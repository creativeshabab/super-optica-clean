Section 01

Frontend Architecture Overview

This project implements a **Hybrid SSR-Micro-Frontend** architecture. PHP handles the heavy structural lifting and SEO pre-rendering, while Vanilla ES13 handles the reactive transactional layer without the overhead of heavy frameworks like React or Vue.

PHP 8.2 Engine Handles templating (header/footer) and secure session handshakes.
Tailwind Build Utility-first JIT compilation for zero-unused CSS in production.
Dynamic Theming Admin settings injected via dynamic_styles.php to :root.

Section 10

Maintenance & Extension

Standards for scaling and extending the platform.

Future Development Rules

  • Sanitization: Never use echo $var in JS script tags; use json_encode($var).
  • Performance: Keep total image payloads under 1.2MB for the index page.
  • Hierarchy: New interactive pages MUST follow the .web-wrapper > .section-padding scaffolding.
  • API Safety: All new endpoints must verify CSRF_TOKEN if not using public API modes.

Lifecycle of an Order

1. Entry: checkout.php collects user address and payment intent.
2. Handle: api/create_razorpay_order.php creates an 'unpaid' row in orders.
3. Commit: On payment success, api/verify_payment.php switches status to 'pending' and commits stock subtraction via PDO Transaction.

Error Mitigation Strategy

A fail-safe alerting system that prevents transaction breakage.

1. Fetch Safeguards

try { const response = await fetch('api/validate.php'); if (!response.ok) throw new Error('Network status: ' + response.status); const data = await response.json(); } catch (error) { showToast("Operation failed: " + error.message, "error"); }

2. Session Filtration

Protected routes are guarded by isLoggedIn(). If a session is lost mid-checkout, the frontend captures the missing token error and redirects to login, preserving the cart state in the database for the user's return.

Performance Strategy

Data Fetching & Optimization

Speed is conversion. We use a hybrid pre-rendering and async model.

Smart Loading Logic

  • Critical Path (SSR): Headers, Navbars, and Hero content are rendered in the first 200ms by PHP.
  • Dynamic Path (AJAX): Appointment slots and Coupon validations are fetched on-demand to keep the page size below 500KB.
  • Image Strategy: All catalog images use loading="lazy" to ensure the DOM is interactive instantly.
Validation Protocol

Advanced Form Handling

Preventing dirty data from reaching the core system.

Validation Pipeline

I. Frontend Filter: Regex check for Phone Numbers (10 Digits Only).
II. Controller Check: PHP filter_var() for emails and htmlspecialchars() for XSS prevention.
III. Schema Lock: MySQL unique constraints on Email/Phone to prevent duplicate account creation.
