1 Plan Your "Delta" Update

Unlike the first deployment, you only need to upload what changed.

  • Did you change PHP code? (e.g., product.php, admin/orders.php)
  • Did you add new CSS/JS? (e.g., assets/css/style.css)
  • Did you add new Database Tables or Columns? (Important!)

2 Upload Changed Files

Using File Manager or FTP, upload the new versions of your files.

Safe to Overwrite:

  • *.php files in root (except config/)
  • assets/ folder (CSS, JS)
  • includes/ folder
  • admin/ folder

‚õî DO NOT TOUCH:

  • config/database.production.php (Never overwrite this!)
  • assets/uploads/ (Unless you are adding new default icons)

3 Database Updates (Migrations)

If you added a new feature (e.g., "Reviews") that needs a new table, you cannot just re-import the whole database. You must apply only the changes.

How to Create a Delta SQL File locally:

  1. Create a new file `sql/update_v2.sql`.
  2. Write only the new `CREATE` or `ALTER` commands.

Example `sql/update_v2.sql`:

-- Add a new column to users table
ALTER TABLE users ADD COLUMN phone_verified TINYINT(1) DEFAULT 0;

-- Create a new table for Wishlist
CREATE TABLE wishlist (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    product_id INT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Applying it on Live Server:

  1. Open PHPMyAdmin -> SQL tab.
  2. Paste the content of your `sql/update_v2.sql`.
  3. Click Go.

4 Maintenance/Testing

After uploading:

  1. Visit the site in Incognito Mode (to clear cache).
  2. Test the New Feature specifically.
  3. Check the "Contact Us" or "Checkout" flow to ensure nothing broke.

Pro Tip: If the site looks broken (CSS issues), clear your browser cache (Ctrl+F5).
