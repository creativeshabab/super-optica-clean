Project Analysis & Architecture

1.1 Technology Stack

Layer Technology
Language PHP 8.x (procedural + some OOP)
Database MySQL via PDO (utf8mb4)
Server XAMPP (Apache) — local dev

1.2 Architecture Rating: 4/10

Monolithic page-based PHP. No MVC framework, no routing layer, no service layer, no dependency injection. Each PHP file independently loads config and includes.

1.3 Code Quality Issues

Issue Severity Description
No MVC/Routing Medium Each page = standalone script. Tight coupling of DB, logic, and view

Phase 5

Security Audit — 10 Vulnerabilities

CriticalSEC-001No CSRF Protection on ANY Form

Affected: ALL forms — login, register, checkout, admin

Attack: External form submits to your site — victim browser sends cookies

Fix: Generate token in session, embed in hidden field, validate on POST

// functions.php — add:
function generateCSRFToken() {
    if (empty($_SESSION['csrf_token']))
        $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
    return $_SESSION['csrf_token'];
}
// Forms: <input type="hidden" name="csrf_token" value="<?= generateCSRFToken() ?>">
HighSEC-002XSS via Unescaped Error Output

File: register.php:L97 — <?= $error ?>

Fix: Use htmlspecialchars($error) everywhere

HighSEC-003OTP Brute-Force — No Attempt Limit

File: verify-otp.php — 6-digit OTP = 1M combos, easily brute-forced

Fix: Limit to 5 attempts per code, cooldown, invalidate after 3 fails

HighSEC-004OTP Resend via GET — No Rate Limiting

File: verify-otp.php:L76 — if (isset($_GET['resend']))

Fix: Use POST, rate-limit 1 per 60s, track count per session

MediumSEC-005Session Fixation — session_regenerate_id missing

Fix: Add session_regenerate_id(true) after successful login

CriticalSEC-006Cart Price Manipulation via Session

File: functions.php:L89, create_razorpay_order.php:L46

Fix: ALWAYS re-fetch prices from DB before creating order

MediumSEC-007File Upload — Extension from User Input

Status: ✓ MIME check works, but extension from $file['name']

Fix: Map MIME → extension server-side

MediumSEC-008Debug/Test Files Exposed in Web Root

Files: debug_variants.php, debug_500.php, check_schema.php, test_variants.php

Fix: DELETE all debug/test scripts before deployment

MediumSEC-009SSL Verification Disabled for SMTP

Fix: Enable peer verification in production; disable only for local dev

HighSEC-010Payment Keys — Ensure Secret Never Leaks

Status: ✓ Debug responses gated by IS_PRODUCTION

Fix: Verify no error response exposes key_secret or paths
